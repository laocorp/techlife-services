────────────────────────────────────
CONTEXTO GENERAL
────────────────────────────────────
Vamos a construir un sistema SaaS llamado provisionalmente “TechLife Service”.

Es un SaaS MULTITENANT, donde múltiples talleres operan de forma aislada y segura dentro de una misma plataforma.

Cada taller puede pertenecer a distintas industrias:

* Automotriz
* Electrónica / Celulares
* Herramientas eléctricas / Maquinaria

El sistema debe ser:

* Escalable
* Seguro
* Modular
* Listo para producción
* Pensado para LATAM

────────────────────────────────────
STACK TECNOLÓGICO OBLIGATORIO
────────────────────────────────────
Frontend:

* Next.js 14+ (App Router)
* TypeScript
* Server Components + Client Components cuando aplique
* TailwindCSS (configurado con diseño corporativo sobrio)

Backend:

* Supabase

  * PostgreSQL
  * Supabase Auth
  * Row Level Security (RLS)
  * Supabase Storage
  * Realtime cuando sea útil

Arquitectura:

* Multi-tenancy con tenant_id en TODAS las tablas
* RBAC por roles
* API Routes de Next.js
* Separación clara de responsabilidades

Deploy pensado para:

* Vercel (frontend)
* Supabase (backend)
